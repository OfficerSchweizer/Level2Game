Defense

by: Bryan Zublin

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I made this type of game because it was one of the first ideas that I thought of while still being relatively easy to create and play.

2. What I Learned

While I was working on the scaling system of the game I mostly just received feedback that the game was either too hard or too easy. As a result of this I added 3 different difficulty settings that affect the speed that difficulty scales by. I also got feedback that I should add a timer to the game so you could see how long you survived for, which I did.

3. Architecture of My Game

My game's setup is very similar to League Invaders' setup because I was taking advantage of some already existing code. Like League Invaders, my project has a GamePanel where main update/draw methods are called based on whether you are in the menu, the actual game, or the game over screen. My other main class is the ObjectManager, which, like League Invaders, stores most of the variables and methods that GamePanel uses. Additionally, I have Shop, Player, Bullet, Base, and Enemy classes that have methods and variables mainly about their function in the game like how they might move around or how they are drawn.

4. Technical Implementation

A part of the game I am proud of is the scaling system which has an integer that the enemies' health, speed, and spawn rate are multiplied by as time goes on. It took a pretty long time balancing this and making sure everything worked.

5. Most Challenging Part

The most challenging part of this project was balancing the scaling system and making sure it wasn't too fast, too slow, and that it was actually possible to play the game for more than 20 seconds on the hardest difficulty. It mostly just came down to trial and error for figuring out how to multiply the main scale integer by the enemies' properties without the properties being buffed too much or too slowly.

6. Future Fixes and Updates

If I was to continue working on this game I would most likely make custom sprites, add separate kinds of weapon types for the player to use, and polish up the simplistic looks of it.
